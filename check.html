Project Overview

Research Objective

Conduct a rigorous empirical comparison between Quantum Machine Learning (QML) models and Classical Machine Learning approaches on a complex Natural Language Processing task.

Dataset & Task

  • GoEmotions Dataset - Google Research multi-label emotion classification
  • 28 emotion labels with severe class imbalance
  • 211,000 training examples, 5,000 validation, 4,000 test

Key Innovation

Standardized evaluation pipeline using BERT-extracted features to ensure fair comparison across all model architectures.

Models Implemented

Quantum-Hybrid Models

  • MLC-QNN: Hybrid Quantum-Classical Neural Network
  • QKSVM: Quantum Kernel Support Vector Machine
  • DRC: Data-Reuploading Classifier

Classical Baselines

  • Fine-tuned BERT: State-of-the-art transformer
  • Classical NN: Direct comparison architecture
  • XGBoost: Gradient boosting ensemble

Technology Stack

PyTorch, PennyLane, Qiskit, Transformers, Scikit-Learn, XGBoost

Research Methodology

Standardized Pipeline

  • Feature Extraction: BERT-base-uncased → 768-dimensional embeddings
  • Data Preprocessing: Consistent splits and label encoding
  • Evaluation Metric: Macro F1-Score (handles class imbalance)
  • Reproducibility: Fixed random seeds, version control

Key Challenges Addressed

  • Severe Class Imbalance: Weighted loss functions and appropriate metrics
  • Computational Constraints: Efficient quantum simulation strategies
  • Quantum-Classical Integration: Seamless gradient flow across frameworks

Technical Implementation

Quantum Circuit Design

  • Parameterized Quantum Circuits (PQCs) using StronglyEntanglingLayers
  • Quantum Feature Maps with AngleEmbedding for classical data encoding
  • Hybrid Training with automatic differentiation across quantum-classical interfaces

Classical Implementation

  • End-to-end BERT Fine-tuning for SOTA baseline
  • Equivalent Architecture Design for fair capacity comparison
  • Advanced Optimization with learning rate scheduling and gradient clipping

Evaluation Framework

Statistical significance testing, per-label analysis, and training dynamics monitoring

Key Research Findings

Performance Analysis

  • Classical models demonstrated superior performance on this NLP task
  • Fine-tuned BERT achieved state-of-the-art results as expected
  • Quantum approaches showed promise but faced current hardware limitations

Technical Insights

  • Computational Bottlenecks: Quantum simulation scales poorly with problem size
  • Feature Transformation: Quantum circuits can learn non-trivial transformations
  • Hybrid Potential: Integration frameworks show promise for future development

Research Contribution

Established rigorous benchmarking methodologies and provided empirical evidence about current QML capabilities and limitations.

Challenges Faced & Solutions

Challenges

  • Conceptual Leap: Classical ML to quantum computing
  • Integration Issues: PyTorch ↔ PennyLane interfaces

Solutions

  • Systematic Learning: Literature review + hands-on practice
  • Debugging Framework: Gradient checking + visualization tools
  • Efficient Sampling: Strategic subset selection
  • Weighted Metrics: Macro F1-Score + pos_weight
